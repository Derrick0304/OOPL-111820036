## 專案簡介

### 遊戲簡介
貓咪大戰爭是一款橫向塔防策略遊戲。玩家在此復刻版中可以透過隨時間自動增加的金錢，派遣最多 10 種各具特色的貓咪士兵，包含基本的「貓咪」、肉盾「坦克貓」、攻擊力強的「戰鬥貓/斧頭貓」、遠程單體「噁心貓」、神速「牛貓」、範圍遠程「鳥貓」、耐打「魚貓」、超遠程單體「蜥蜴貓」、強大範圍「巨神貓」。目標是抵擋不斷來襲的各色敵方角色（如狗、蛇、河馬、豬等），並最終摧毀位於左側的敵方基地。
本復刻版本基於 C++ 與 PTSD 2D 遊戲引擎開發，重現了原作的主選單、章節切換、選關滑動、出兵冷卻、金錢上限與生產速度升級、貓咪砲發射機制（包含雷射與連環爆炸傷害波）、戰鬥結算（XP）等核心系統。

## 遊戲介紹

### 遊戲規則
1. **出兵系統**：畫面上方顯示當前金錢與上限。玩家點擊下方已裝備的貓咪按鈕，若金錢足夠且該貓咪冷卻結束，即可扣除金錢並從己方基地（右側）召喚貓咪。
2. **升級錢包**：點擊「工作貓」升級按鈕可花費金錢提升金錢增長速度以及金錢上限。
3. **貓咪砲**：當右下角貓咪砲能量蓄滿後，點擊可發射貓咪砲。雷射波會從己方基地向左掃描，對路徑上的所有敵方單位造成固定傷害並強制擊退。
4. **勝負判定**：
   - 己方基地（位於右側）血量歸零 -> 失敗 (DEFEAT)。
   - 敵方基地（位於左側）血量歸零 -> 勝利 (VICTORY)。
5. **升級與編隊**：戰鬥獲勝可獲得 XP 獎勵。回到主選單後可進入 `Upgrade` 介面使用 XP 提升貓咪等級，或進入 `Equip` 介面自由調整上場的 10 隻貓咪隊伍。

### 遊戲畫面
1. **開始畫面 (StartScene)**：提供 Play 按鈕。
2. **主選單 (MainMenuScene)**：包含 Start、Upgrade、Equip 按鈕與 XP、貓罐頭動態顯示。
3. **章節輪播與選關畫面 (StageSelectScene)**：可滑動拖曳之地圖背景，關卡路線，支援滑鼠拖曳、自動定位彈回。
4. **戰鬥畫面 (BattleScene)**：包含基地、貓咪/敵人走路與攻擊動畫、發射貓咪砲的雷射效果與下方的出兵欄位。

## 程式設計

### 程式架構
專案使用物件導向程式設計（OOP）與場景管理員模式：
- `App`：主程式控制器，管理全域變數（如 XP、罐頭、貓咪等級、裝備隊伍）與延遲場景切換 (`ChangeScene`)。
- `IScene`：場景基礎介面，定義 `Enter()`、`Update()`、`Exit()` 與 `GetRoot()`。
- `MainMenuScene`、`StageSelectScene`、`BattleScene`、`UpgradeScene`、`EquipScene`、`StartScene`、`ChapterSelectScene`：繼承自 `IScene`，分別管理各自的畫面與邏輯。
- `Unit`：代表戰鬥單位的類別，包含血量、攻擊力、射程、速度等狀態機（走路、攻擊、擊退、死亡）。
- `UnitManager`：管理所有場上的戰鬥單位，執行每影格的碰撞檢測、傷害結算、貓咪砲判定、單位死亡清理。
- `UnitFactory`：單例工廠，從 `Units.json` 中讀取並註冊所有單位屬性，並負責生成實體與等級數值加成（每級 HP 與 ATK 加成 10%）。
- `StageLoader` / `WaveSpawner`：負責從 `Stages.json` 中讀取關卡與怪獸刷怪波次設定，在戰鬥中定時或根據基地血量刷出敵人。
- `UIManager`：整合戰鬥中的 UI 元件，如出兵按鈕 `UnitButton`、升級錢包按鈕 `WorkerCatButton`、貓咪砲按鈕 `CatCannonButton` 等。

### 程式技術
1. **資料與程式邏輯分離 (Data-Driven Design)**：
   - UI 元件座標外置於 `UI_Layout.json` 中。
   - 所有單位屬性外置於 `Units.json` 中。
   - 所有關卡資料與怪獸刷怪波次外置於 `Stages.json` 中。
   - 所有的地圖點位座標外置於 `Map_Coordinates.json` 中。
2. **碰撞阻擋系統 (Visual Frontline Alignment)**：
   - 計算雙方寬度，以 `x - width` (CAT) 與 `x + width` (ENEMY) 計算出精確的身體前緣。
   - 使用 `realDist = frontA - frontB` 作為實質視覺 gap。
   - 修正了 frame steps 帶來的穿透 Bug：將碰撞邊界檢測從原先的 `realDist >= 0` 放寬至 `realDist >= -std::max(widthA, widthB)`，當兩者略微重合時仍能觸動阻擋與戰鬥，防止單位互相穿透過走。
3. **效能優化 (Pre-baking Static Textures)**：
   - 由於原版地圖包含 48 個關卡紅點與折線，動態加載上百個點線 GameObject 會在場景切換時造成 CPU 瓶頸。
   - 開發 `Map_Coords_Collector.py` 腳本，在點位儲存時，自動利用腳本將所有連接路徑與縮小為一半大小的紅白點直接「烘焙」到背景 PNG `StageSelect.png` 上，將動態節點渲染數降為 0。
4. **音訊控制器設計 (BGM Controller)**：
   - 實作防重複播放的 `PlayBGM` 路由，僅在曲目切換時調用 `BGM::Play`，避免選單場景切換時音樂中斷。

## 結語

### 問題與解決方法
1. **問題：在關卡選擇介面，切換進去時會產生約 1.5 秒的黑屏卡頓。**
   - *原因*：因為動態繪製地圖上 48 個關卡點與連接路徑時，需要實例化近 300 個 GameObject，造成渲染樹劇烈抖動。
   - *解決方法*：腳本在儲存座標的同時，將線條與縮小一半的點預先烘焙至地圖背景 PNG，C++ 只需加載單張大背景，耗時降至 0 秒。
2. **問題：戰鬥中近戰貓咪（如基本貓、坦克貓、巨神貓）會在距離敵人很遠的地方開始揮拳，像是在打空氣。**
   - *原因*：直接使用了 Wiki 上的射程單位（例如 140），在 C++ 像素座標系中對應 140 像素的間距太大。
   - *解決方法*：將射程轉像素比例調整為 `0.2f`，並將碰撞檢測條件改為支援負值（允許略微重合），使角色在動畫發動時正好頭部或拳頭能打在對方身上。
3. **問題：當大量高移動速度單位（如牛貓）衝鋒時，有時會發生與敵方單位「擦肩而過」直接穿透走到基地底部的 bug。**
   - *原因*：前緣距離計算採用 `realDist >= 0`。若某影格中兩者速度過快，直接走到 `realDist = -3` 的重疊狀態，會因條件不符而判斷為「未接觸」並繼續 WALK。
   - *解決方法*：放寬檢測下限為 `-std::max(widthA, widthB)`。
