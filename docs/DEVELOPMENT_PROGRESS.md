# 貓咪大戰爭 - 開發進度報告 (Development Progress)

## 1. 目前開發狀態 (截止至 2026/03/13 11:15)
專案進度持續領先，已完成 **Week 10** 的全部目標並進入 **Week 11-12** 的功能擴充。本階段重點在於**素材自動化處理**、**數據驅動深化**以及**視覺表現優化**。

---

## 2. 已完成功能 (Completed Features)

### 素材與單位擴充 (New!)
*   **牛貓 (CowCat)**：高移動速度 (80)、高攻擊頻率 (0.3s) 的快攻型單位。
*   **鳥貓 (BirdCat)**：長射程 (170)、高傷害 (250) 的後排輸出單位。
*   **素材自動化處理**：優化了 `split.py` 與整合流程，所有素材（含 Icon）皆經過「貼身裁切」，確保 UI 對齊與 Pivot 計算一致。

### 視覺與核心邏輯優化 (New!)
*   **高度偏移系統 (yOffset)**：在 `UnitData` 與 JSON 中加入 `yOffset`，實現如 BirdCat 的**浮空視覺效果**，且不影響底部的碰撞判定。
*   **動態 Pivot 對齊**：實作每幀動態更新 Pivot 於圖片底部，解決不同幀動畫尺寸不一導致的跳動問題。
*   **攻擊動畫保護機制**：
    *   修正了打死敵人後動畫被「腰斬」的 Bug，單位現在會完整做完目前的攻擊動作才恢復行走。
    *   優化 `SetState` 邏輯，攻擊動畫改為一次性播放 (`looping = false`) 以精確控制狀態切換。

### 數據外部化 (Deep Implementation)
*   **JSON 配置系統**：所有單位屬性（HP, ATK, Speed, Range, Cost, Cooldown, yOffset, 幀數）皆由 `Resources/Data/Units.json` 驅動。

---

## 3. 程式架構說明 (Architecture)

*   **`Unit` 類別**：新增 `m_YOffset` 屬性，並在 `Update` 中動態修正 `m_Pivot`。
*   **`UnitFactory`**：解析 JSON 並負責根據數據註冊所有貓咪與敵人原型。
*   **`UnitButton`**：自動適配 `icon.png` 的 `GetScaledSize()`，確保 UI 遮罩與冷卻條對齊。
