# 貓咪大戰爭 - 開發進度報告 (Development Progress)

## 1. 目前開發狀態 (截止至 2026/03/12)
目前專案已完成 **Week 3 至 Week 6** 的核心目標，並提前實作了 **Week 10** 的基地原型。遊戲已具備基礎的戰鬥循環與場景移動功能。

---

## 2. 已完成功能 (Completed Features)

### 核心戰鬥機制
*   **單位管理 (UnitManager)**：實作了模組化的管理器，負責處理所有單位的生成、碰撞偵測、傷害計算與死亡清理。
*   **狀態機系統**：單位具備 `WALK` (行走)、`ATTACK` (攻擊)、`DEAD` (死亡) 三種狀態，並會根據狀態自動切換動畫。
*   **傷害與冷卻**：實作了基於時間 (`DeltaTime`) 的攻擊冷卻機制，確保單位按特定頻率互相扣血。
*   **陣營系統**：支持 `CAT` (貓咪) 與 `ENEMY` (敵人) 兩大陣營，具備不同的移動方向與敵我識別邏輯。

### 視覺與動畫
*   **幀動畫支援**：全面採用 `Util::Animation`，支持多張圖片組成的序列幀動畫。
*   **素材整合**：已完成背景 (`background.png`)、基地 (`cat_base`, `enemy_base`) 與角色走路/攻擊動畫的整合。
*   **圖層管理 (Z-Index)**：建立了明確的渲染層級（背景 -10 > 基地 -1 > 小兵 1），確保視覺遮擋正確。

### 場景與攝影機
*   **虛擬攝影機捲動**：透過移動世界根物件 (`m_Root`) 實作了畫面左右捲動功能（鍵盤方向鍵控制）。
*   **比例精確化**：根據圖片原始像素尺寸 (PNG Metadata) 計算並設定了最適合的縮放比例與地平線位置。

---

## 3. 程式架構說明 (Architecture)

*   **`App`**：最高層級，負責監聽全局輸入（捲動、測試出兵）與驅動管理器。
*   **`UnitManager`**：戰鬥邏輯核心，負責處理 `Unit` 之間的互動。
*   **`Unit` (繼承自 GameObject)**：所有戰鬥單位的基類，封裝了屬性、狀態與動畫切換。
*   **`Tower` (繼承自 Unit)**：基地的特殊實作，具備高血量且靜止不動。
