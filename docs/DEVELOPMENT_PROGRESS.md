# 貓咪大戰爭 - 開發進度報告 (Development Progress)

## 1. 目前開發狀態 (截止至 2026/03/13)
目前專案已大幅超出預期進度，完成了 **Week 9 至 Week 10** 的核心 UI 與金錢系統，並提早實作了 **Week 11** 的數據驅動架構。遊戲目前具備完整的戰鬥循環、出兵冷卻與基礎 AI 敵軍生成。

---

## 2. 已完成功能 (Completed Features)

### 核心戰鬥與數據架構
*   **數據驅動工廠 (UnitFactory)**：實作了 `UnitData` 結構，將單位數值（HP, ATK, 射程, 成本, 冷卻）抽離。目前支援 `BasicCat`, `TankCat`, `AxeCat`, `GrossCat` 與 `Doge`。
*   **自動化資源載入**：工廠會根據路徑自動偵測並載入 `Walk` 與 `Attack` 的序列幀動畫。
*   **單位管理 (UnitManager)**：處理所有單位的生成、碰撞偵測、傷害計算、死亡清理及基地血量監控。
*   **狀態機系統**：單位具備 `WALK` (行走)、`ATTACK` (攻擊)、`DEAD` (死亡) 三種狀態。

### 金錢與 UI 系統 (提前完成)
*   **自動理財系統**：實作隨時間增加金錢（預設每秒 50 元），並與出兵邏輯連結。
*   **互動式按鈕 (UnitButton)**：
    *   **冷卻機制**：按鈕具備藍色進度條顯示出兵冷卻。
    *   **金錢檢測**：餘額不足時按鈕會自動顯示黑色遮罩並停用。
    *   **動畫回饋**：成功出兵時按鈕會產生縮放震動效果。
*   **UI 資訊顯示**：畫面左上方即時顯示當前金額。

---

## 3. 程式架構說明 (Architecture)

*   **`App`**：遊戲主迴圈，負責驅動管理器、處理金錢增長與簡單的敵軍 AI 出兵邏輯。
*   **`UnitFactory`**：單例工廠，負責註冊所有單位的原型數據並生成實例。
*   **`UIManager`**：管理所有 `UnitButton` 並透過回調 (Callback) 與 `App` 的金錢系統溝通。
*   **`Unit` / `Tower`**：戰鬥個體基類，封裝了屬性、狀態與動畫切換邏輯。
