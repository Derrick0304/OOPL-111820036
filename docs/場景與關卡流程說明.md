# 場景與關卡流程說明

## 目前場景流程

目前程式執行流程如下：

1. 主選單 `MainMenuScene`
2. 選關畫面 `StageSelectScene`
3. 戰鬥畫面 `BattleScene`
4. 戰鬥結束後於 `BattleScene` 內顯示結果選單

結果選單目前提供：

- Retry
- Back to Stage Select
- Back to Main Menu

## 場景切換方式

場景切換由 `App` 統一管理。

目前 `App` 不會在按鈕 callback 觸發當下立刻切換 scene，而是先記錄成 pending scene，等當前 update 結束後再正式套用。

這樣做的原因是避免：

- scene 在自己的 `Update()` 執行途中被替換
- 舊 scene 物件在 callback 中被提早釋放
- 發生切場景後程式直接關閉或崩潰的問題

## 各模組責任

### `App`

位置：

- `include/App.hpp`
- `src/App.cpp`

責任：

- 啟動時載入單位資料與關卡資料
- 持有目前 scene
- 管理 deferred scene switching
- 每幀呼叫 scene 的 `Update()`
- 呼叫 renderer 繪製畫面

### `IScene`

位置：

- `include/Scene/IScene.hpp`

責任：

- 定義各 scene 共用介面
- 目前包含：
  - `Enter()`
  - `Update()`
  - `Exit()`
  - `GetRoot()`

### `MainMenuScene`

位置：

- `include/Scene/MainMenuScene.hpp`
- `src/Scene/MainMenuScene.cpp`

責任：

- 顯示主選單背景
- 顯示 `Start` 與 `Exit`
- `Start` 切到選關畫面
- `Exit` 結束程式

### `StageSelectScene`

位置：

- `include/Scene/StageSelectScene.hpp`
- `src/Scene/StageSelectScene.cpp`

責任：

- 顯示選關背景
- 讀取已載入的 stage 清單
- 以文字按鈕列出所有關卡
- 點擊關卡後進入對應 `BattleScene`
- 提供返回主選單

### `BattleScene`

位置：

- `include/Scene/BattleScene.hpp`
- `src/Scene/BattleScene.cpp`

責任：

- 根據指定 stage 建立戰鬥場景
- 建立背景、基地、`UnitManager`、`UIManager`
- 管理金錢成長
- 驅動 `WaveSpawner`
- 檢查勝負
- 顯示戰鬥結果選單

## 關卡資料

關卡資料檔位置：

- `Resources/Data/Stages.json`

目前每個關卡支援欄位：

- `id`
- `displayName`
- `background`
- `enemyBaseHp`
- `waves`

### `waves` 結構

目前 `waves` 是時間驅動的波次設定，每個 wave 包含：

- `triggerTime`
- `spawns`

每個 spawn 包含：

- `unit`
- `count`
- `interval`

範例：

```json
{
  "id": "stage_001",
  "displayName": "First Battle",
  "background": "/Backgrounds/background.png",
  "enemyBaseHp": 1000.0,
  "waves": [
    {
      "triggerTime": 2.0,
      "spawns": [
        { "unit": "Doge", "count": 1, "interval": 0.0 }
      ]
    }
  ]
}
```

## 關卡資料載入

### `StageLoader`

位置：

- `include/Stage/StageLoader.hpp`
- `src/Stage/StageLoader.cpp`

責任：

- 載入 `Stages.json`
- 解析為 `StageData`
- 提供 stage 清單給選關畫面與戰鬥畫面使用

### `StageData`

位置：

- `include/Stage/StageData.hpp`

責任：

- 定義關卡資料結構
- 定義 wave 與 spawn 的資料格式

## 敵人波次生成

### `WaveSpawner`

位置：

- `include/Battle/WaveSpawner.hpp`
- `src/Battle/WaveSpawner.cpp`

責任：

- 追蹤戰鬥經過時間
- 檢查哪些 wave 應該被觸發
- 把 wave 轉成實際的 spawn job
- 按 `interval` 逐次生成敵人

這樣做的好處是把：

- 關卡設定
- 波次觸發
- 敵人生成

從 `App` 與 `BattleScene` 的主流程中拆開，後續擴充會比較清楚。

## 目前限制

目前系統仍有以下限制：

- 主選單與選關仍是暫時性的文字 UI
- 可用角色清單尚未跟關卡綁定
- 金錢規則尚未外部化到 stage
- 波次條件目前只有 `triggerTime`
- 選關畫面尚未提供關卡描述或預覽資訊

## 後續建議方向

後續可以往這些方向擴充：

- 讓 `Stages.json` 支援更複雜的波次條件
- 加入 stage-specific available units
- 加入 stage-specific economy rules
- 將臨時文字 UI 替換成正式選單介面
- 在選關畫面加入關卡描述、預覽圖與其他 metadata
